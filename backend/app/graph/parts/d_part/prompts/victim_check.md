# 미인지형 판별 프롬프트 (초안)

사용자의 상황 서술을 아래 전세사기피해자법 요건 슬롯에 매핑하세요.

1. 전입신고 + 확정일자 여부
2. 보증금 5억 이하 여부 (최대 7억 상향 가능)
3. 2인 이상 피해(예상) 여부
4. 반환의도 없음을 의심할 상당한 이유

각 슬롯에 대해 [충족 / 불충족 / 불명확] 중 하나로 판정하고, 근거를 함께 제시하세요.
아래 "응답 형식"에 정의된 filled/unfilled/unclear 세 값 외에 다른 값(예: 높음/있음/추가확인 같은
최종판단 어휘)은 슬롯 값으로 절대 쓰지 마세요 — 그건 이 프롬프트가 다루는 범위가 아닙니다.

**이번 발화에서 새로 확인된 것만 판정하고, 확인되지 않은 것은 "unclear"로 두세요.** 기존 슬롯 값이
함께 주어지더라도 그건 대화 맥락을 이해하기 위한 참고 자료일 뿐입니다 — 기존 값을 그대로 베껴
넣지 말고, 이번 발화만 놓고 판정하세요. 기존 값과의 병합은 시스템이 알아서 처리합니다.

경공매(공매/경매)가 이미 완료됐다는 언급이 있으면 auction_completed를 true로 판정하세요.
**언급이 없으면 false가 아니라 null입니다** — 이번 발화에서 확인되지 않았다는 뜻이지, 완료되지
않았다는 뜻이 아닙니다.

## 응답 형식

아래 JSON 형식으로만 응답하세요. 다른 텍스트를 덧붙이지 마세요. 판정 못한 슬롯은 "unclear"로 표기하세요.

```json
{
  "moved_in_and_fixed_date": "filled",
  "deposit_under_limit": "unfilled",
  "multiple_victims": "unclear",
  "no_intent_to_return": "filled",
  "multiple_victims_reason": "근거 텍스트 또는 null",
  "auction_completed": null
}
```
